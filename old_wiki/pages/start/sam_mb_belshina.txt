====== САМ МБ БЕЛШИНА ======

**Остатки **у данного поставщика обновляются **автоматически.**

**Цены **приходят отдельно на почту:

**Кроме Белшины-никакие позиции не  обрабатываем** 
{{ :start:сам_мб_белшина.png?1000 }}**Входная цена** - ЦЕНА с НДС, руб. (ТРАНЗИТ). У данного поставщика есть наценки. (см. алгоритм)

Для обновления цен **"САМ МБ БЕЛШИНА" **на ресте существует отдельный скрипт

{{ :start:белшина_1.png?200 }}{{ :start:белшина_2.png?400 }}В этот скрипт загружается общий файл, где будет **СХСП **и **ГРУЗ**.** Наценки для СХСП и ГРУЗ могут отличаться **

Я обычно формирую файл **"Цены ВСЕ"** в котором в начале файла идет СХСП, а ниже ГРУЗ

Пример:

{{:start:сам_мб_белшина_пример.xlsx}}

{{ :start:белшина_3.png?500 }}

**НО ПРЕЖДЕ ЧЕМ ОБНОВЛЯТЬ ЧЕРЕЗ СКРИПТ НУЖНО: **Сформировать еще 2 файла для **СХСП **и **ГРУЗ**, где будет только **АРТИКУЛ** и **НОМЕНКЛАТУРА. **Эти файлы нужно прогнать через обычное обновление, без обнуления, по всем складам БЕЛШИНЫ, от ЕКБ до Электроуглей.
 Это нужно для того, что бы номенклатура обновилась и новые позиции из прайса попали в нашу БД

После всего нужно прожать **Автообновление - САМ МБ БЕЛШИНА**

=====МИГРАЦИЯ ЦЕН=====
**НЕ** выбираем галочку в миграции ( Очищать старые цены? ) у Сам Мб Белшина. 
Так как она затирает цены из 1С, для которых нет цен миграции.

Если нет остатка у товарного предложения от поставщика, то и цен для миграции на наши склады не будет и из 1С цены будут затираться (  если поставить галочку )
