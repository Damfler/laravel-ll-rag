  * **СТРУКТУРА ВСЕХ КЛАССОВ**
$arElems = $this-> jsonInfo (); - Получает основной массив от поставщика.
Получает она либо на прямую, либо сохраняет данные в файл и через decode преобразует к массиву. Причем массив может быть сразу и для шин, и для дисков. Если так, то формируется массив с данным для необходимой категории и через return отправляется к следующей функции.
В некоторых случаях начинает формироваться общая структура массива для всех (имя, артикул, тип).
$arElems = $this-> updateInfo ($arElems); - Преобразует массив поставщика к общему по структуре: Массив => [NAME | ARTNUMBER | CAE | TYPE | STOCKS => [price | price_rozn | rest | whr | provider_id]].
После получения массива поставщика он принимает форму, описанную выше, заполняет все остатки и так же через return отправляется к третьей функции. 
На этом этапе, через id склада поставщика (whr) ищется id нашего склада в бд, и записывается в provider_id, для последующей эксплуатации.
$arElems =$this-> saveResult ($arElems); - Сохраняет данные в базу данных
Для начала очищает все остатки поставщиков, записывает в бд об очистке. Так же изменяется время обновления у поставщика на актуальное.
Далее выбираются все товары с бд, и сверяются с товарами от поставщика, если товар найден обновляют остатки, если же нет, то товар добавляется в бд. Если это диски, то их параметры добавляются в отдельную таблицу.
Под конец так же создается лог об обновление остатков и задача, что было обновление.

run (); - Прогоняет все выше указанные циклы едино.

Если диски, то дополнительно берется информация о бренде, модели, цвете, типа диска, ширине, диаметре, диа, ет, псд (либо в первой, либо во второй функции).
Именно вторая функция приводит массив поставщика к общему виду, чтобы функцию сохранения в бд можно было использовать везде.

В таблице ‘providers’ есть поле ‘outer_id’, это id склада поставщиков. Он всегда ищет id провайдеров через это поле.

CRON файл находится по пути: /auto_update/index.php

  * **4ТОЧКИ**
Тип выгрузки: JSON + API
Путь к файлам: /auto_update/4tochki/
Название классов: FourTochkiTires, FourTochkiRims

4Точки находятся в 3-х городах, под каждый город своя учетная запись со своими данными (пароль и логин от api, выгрузка), причем выгрузка настраивается вручную, а данные от api создаются автоматически.
Файл с выгрузкой дается раз в день, так что мы сохраняем данный файл на сервер, на день (если время создания больше 24 часов, то он заменяется на новый файл).
Функционал API 4точек позволяет только получать информацию о заказах, их статусе и необходимую нам, информация о товаре (стоимость, на каких складах находится, сколько в наличие).

В первой функции формируется массив с артикулами и названиями предложений со всех городов (сохраняется 3 файла, каждый под свой город).
Так же берется основная информация для дисков (в структуре всех классов описано какие поля).

Во второй функции он обращается через api к 4точки. В запросе участвуют поля логин, пароль и массив с артикулами предложений (не больше 2000).
API возвращает массив с артикулам товара, стоимостью (входной и розничной), id склада поставщика и кол-во товара на данном складе.
ID складов берется на официальном сайте.
В этой же функции по группе поставщика или id их склада выбираются id склада у нас в бд.

В третьей функции все сохраняется в бд по стандарту.

  * **АВТОЭКСПЕРТ**
Тип выгрузки: YML + XML
Путь к файлам: /auto_update/autoExpert/
Название классов: AutoExpertTires, AutoExpertRims

В YML находятся все id товарных предложений определенной категории (диски, шины). В XML же цены и все склады, на которых ТП есть.

В первой функции формируется массив с id ТП из YML (сохраняется в файл, для чтения).

Во второй функции он обращается к XML файлу (сохраняется в файл, для чтения). Дальше перебором выбираются по id все данные и формируется к общему массиву (там же и id склада).
В этой же функции по группе поставщика или id их склада выбираются id склада у нас в бд.

В третьей функции все сохраняется в бд по стандарту.

  * **БРИНЕКС**
Тип выгрузки: JSON
Путь к файлам: /auto_update/brinex/
Название классов: BrinexTires, BrinexRims, BrinexMoto

JSON файл находится на сервере, он сам автоматически создается каждое определенное время, по пути /ftp_provider/brinex/brinex.json. В нем находятся вся информация: и о шинах, и о дисках, о мото (в данной выгрузке идет мото + квадро).

В первой функции из файла берется информация только о ТП категории (либо шина, либо диск) и передается во вторую функцию.

Во второй функции формируется общий массив, по данным с первой функции.
В этой же функции по группе поставщика или id их склада выбираются id склада у нас в бд.

В третьей функции все сохраняется в бд по стандарту.

  * **НАШИ СКЛАДЫ**
Тип выгрузки: JSON
Путь к файлам: /auto_update/our_w/
Название классов: OurStorage

Вся выгрузка берется от 1С. Используются 3 ссылки (список групп, список товаров с остатками, список складов).

Ссылка список складов используется только для получения outer_id складов в нашей бд.

В первой функции выбираются код складов из определенных списков групп (зима, лето, диски). Дальше по ссылке списка товаров с остатками добавляется код склада и получаем все данные о ТП с этого склада.

Во второй функции формируется общий массив, по данным с первой функции. В этой же функции по группе поставщика или id их склада выбираются id склада у нас в бд.

В третьей функции все сохраняется в бд по стандарту.

Путь к файлам: /auto_update/severAuto/

Файл выгрузки выдается каждые 10 минут. Так что если файл существует больше 10 минут или его нет, его создают и заливают на сервер

В первой функции сохраняется и открывается файл. Получаем массив от производителя и передаем во вторую функцию.

Во второй функции формируется общий массив, по данным с первой функции. В этой же функции по группе поставщика или id их склада выбираются id склада у нас в бд.

В третьей функции все сохраняется в бд по стандарту.

  * **ШИННЫЕ РЕШЕНИЯ. РОСТОВ**
Тип выгрузки: XML
Путь к файлам: /auto_update/tyreSol/
Название классов: BusSolutionsTires, BusSolutionsRims

В файле присутствует выгрузка всех категорий. В основном там 3 массива, это products – все ТП, rest – склады и наличие, prices – стоимость. Общее внутри всех этих массивов cae коды.

В первой функции сохраняется и открывается файл. Получаем массив от производителя и сразу формируется массив, а точнее из всех 3-х массивов внутри выбирается определённая категория и записывается в отдельный массив, после этого передается во вторую функцию.

Во второй функции формируется общий массив и добавляется id провайдера, отправляется на сохранение.

В третьей функции все сохраняется в бд по стандарту.
